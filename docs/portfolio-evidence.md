# Portfolio Evidence

This repository translates the AI-based Local-First Marketing Agent into public software engineering evidence without exposing the private implementation.

## What this project demonstrates

- **AI product architecture:** an end-to-end marketing operating system rather than an isolated content generator.
- **Agentic workflow design:** natural-language control, campaign orchestration, structured post packages, MCP integration, and governed sidecar task management.
- **Local-first engineering:** local workspace state, local or compatible LLM execution, local media and approval records, and controlled external calls.
- **Human governance:** approval gates, dry-run-first publishing, connector maturity controls, and account assignment safeguards.
- **Campaign intelligence:** product-growth ranking, landing-page briefs, simulation preflight, analytics import, lessons extraction, and learned scoring.
- **Integration thinking:** YouTube OAuth and channel governance, guarded social scaffolds, website/FTP inventory, MiroFish simulation, Paperclip control plane, and external LLM orchestration.
- **Operational maturity:** tests, linting, frontend build verification, health checks, audit records, scheduler idempotency, and observability artifacts.

## Why this is stronger than a code dump

A public source dump would expose proprietary logic, connector behavior, prompt structures, credentials, and trade-secret implementation detail. This repository instead provides a portfolio evidence layer:

- architecture diagrams;
- workflow descriptions;
- operating model;
- security boundaries;
- demo screenshots;
- demo video;
- quality and testing evidence;
- implementation constraints and maturity notes.

It shows the ability to design and deliver a serious AI software system while protecting the private implementation.

## Interview talking points

- How to separate creative AI generation from governed marketing execution.
- Why external LLMs should operate through MCP allowlists instead of direct credential access.
- How to build approval-gated publishing with idempotent scheduler execution.
- Why simulation should be treated as synthetic preflight and analytics as source of truth.
- How to structure multi-brand workspaces to prevent wrong-account publishing.
- How campaign Autopilot avoids duplication by using one dossier with many topic bundles.
- How product-growth analysis connects marketing content with commercial prioritization.
- How optional sidecars such as Paperclip can govern agents without owning platform credentials.
